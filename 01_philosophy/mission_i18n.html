Why Tengri?

Tengri is a modern, high-performance programming language inspired by the agglutinative morphology of the Kazakh language.

Our mission is to turn linguistic clarity into computational clarity. We aim to combine expressive minimalism with predictable performance and safety, following a single semantics across AST → VM → JIT → AOT. The project is international and open by default.

Design Principles

  • Expressive minimalism: a small, orthogonal core with powerful patterns.
  • Determinism: visible costs, predictable evaluation.
  • Safety first: explicit mutation, clear lifetimes, simple errors.
  • International by default: Unicode-ready, inclusive docs and tools.

Performance Roadmap

  1. AST Interpreter — correctness, REPL.
  2. Register VM — lean bytecode, peephole passes, constant folding.
  3. Tiered JIT — fusing hot paths, inlining, bounds elimination.
  4. AOT Compiler — SSA IR, PGO, link-time specialization.

What success looks like

  • Productivity of Python/JS with near-native speed on kernels.
  • Transparent benchmarks in CSV/plots, reproducible scripts.
  • Seamless FFI with C/Go/Rust; easy embedding.

Почему Tengri?

Tengri — современный высокопроизводительный язык, вдохновлённый агглютинативной морфологией казахского языка.

Наша цель — превратить лингвистическую ясность в вычислительную. Мы объединяем выразительный минимализм с предсказуемой производительностью и безопасностью, сохраняя единые семантики на этапах AST → VM → JIT → AOT. Проект изначально международный и открытый.

Принципы дизайна

  • Выразительный минимализм: небольшой ортогональный ядро и мощные паттерны.
  • Детерминизм: видимая стоимость, предсказуемое исполнение.
  • Безопасность прежде всего: явная мутация, понятные времена жизни, простая обработка ошибок.
  • Интернациональность по умолчанию: Unicode, инклюзивная документация и инструменты.

Дорожная карта производительности

  1. AST-интерпретатор — корректность, REPL.
  2. Регистровая VM — байткод, peephole-пассы, свёртка констант.
  3. Многоуровневый JIT — склейка горячих путей, inline, устранение проверок.
  4. AOT-компилятор — SSA IR, PGO, специализация на этапе линковки.

Как выглядит успех

  • Продуктивность Python/JS при скорости, близкой к нативной, на ядрах.
  • Прозрачные бенчмарки: CSV/графики, воспроизводимые скрипты.
  • Простой FFI с C/Go/Rust; лёгкое встраивание.

Неліктен Tengri?

Tengri — қазақ тілінің агглютинативті морфологиясынан шабыттанған жоғары өнімді бағдарламалау тілі.

Мақсатымыз — лингвистикалық анықты есептеулік айқындыққа айналдыру. Біз AST → VM → JIT → AOT кезеңдерінде бірыңғай семантиканы сақтай отырып, экспрессивті минимализмді болжамды өнімділік және қауіпсіздікпен біріктіреміз. Жоба бастаптан халықаралық және ашық.

Дизайн қағидаттары

  • Экспрессивті минимализм: шағын өзек және қуатты үлгілер.
  • Детерминизм: көрінетін құн, болжамды орындау.
  • Қауіпсіздік алдымен: айқын мутация, өмірлік циклдер, қарапайым қателер.
  • Әлемдік бағыт: Unicode, инклюзивті құжаттама және құралдар.

Өнімділік жол картасы

  1. AST интерпретаторы — дұрыстық, REPL.
  2. Регистрлік VM — байткод, peephole-пасстар, константаларды бүктеу.
  3. Көпдеңгейлі JIT — «ыстық» жолдарды біріктіру, inline, тексерулерді қысқарту.
  4. AOT компиляторы — SSA IR, PGO, link-time мамандандыру.

Табыстың өлшемі

  • Python/JS деңгейіндегі өнімділік, ядроларда нативке жақын жылдамдық.
  • Ашық бенчмарктер: CSV/графиктер, қайталанатын скрипттер.
  • C/Go/Rust-пен оңай FFI; қарапайым ендіру.
